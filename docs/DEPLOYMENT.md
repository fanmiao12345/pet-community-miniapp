# 部署与回滚

## 1. 部署前准备

- 微信小程序 AppID 与符合业务范围的主体/类目
- CloudBase 环境 ID
- 已执行 `npm run verify`
- 已填写正式隐私政策与用户协议
- 已确认内容安全消息推送地址

## 2. CloudBase 部署顺序

1. 创建 `docs/CLOUD_SETUP.md` 列出的集合与索引。
2. 部署 `auth`、`user`、`pet`。
3. 部署 `contentSecurity`、`content`、`moderationCallback`。
4. 部署 `interaction`、`governance`、`admin`。
5. 初始化话题。
6. 将指定用户的 `role` 改为 `admin`。
7. 配置异步内容安全回调。
8. 将 `ENV.dataMode` 切换为 `cloud` 并填写环境 ID。

## 3. 小程序发布

1. 微信开发者工具执行“构建 npm”（项目当前无运行时 npm 依赖时可跳过）。
2. 清缓存后完整回归 `docs/TEST_CASES.md`。
3. 上传体验版，完成双账号与真机测试。
4. 核对隐私保护指引和接口调用声明。
5. 提交审核并记录版本号、提交时间和云函数版本。

## 4. 监控基线

至少监控：

- 云函数错误率与 P95 时延
- 数据库读写量和存储量
- 云存储容量与外网流量
- pending 审核任务数量与最长等待时间
- 举报积压和管理员处理时长
- 用户登录失败、发布失败和内容安全接口异常

## 5. 回滚

### 小程序端

保留上一个稳定版本，在微信公众平台使用版本回退。

### 云函数

- 发布前保存上一版本代码包或 Git 标签。
- 回滚时优先恢复云函数，再回滚小程序端。
- 数据结构新增字段应保持向后兼容，避免旧版本读取失败。

### 数据

- 高风险批量操作前导出集合。
- 运营后台仅执行软删除和状态更新。
- 不直接批量删除用户生成内容，除非有备份与明确审批。


## 手机号登录部署

- 使用具备手机号能力的真实小程序 AppID，不能使用游客 AppID 验收真实授权。
- 为 `auth` 设置 `PHONE_HASH_SECRET`，不得提交到代码仓库。
- 部署 `auth/config.json` 中的 `phonenumber.getPhoneNumber` OpenAPI 权限。
- 扫码进入动态详情、话题详情和用户主页，分别验证授权后的回跳。
- 发布前在小程序管理后台完成隐私保护指引，明确手机号用途、处理方式和注销渠道。
