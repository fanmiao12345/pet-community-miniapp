# UI 设计说明（v1.2.0）

本轮在不改变业务逻辑和数据接口的前提下，对核心用户路径进行了统一视觉升级。

## 设计原则

- 温暖但不过度幼稚：以珊瑚橙作为主色，薄荷绿作为社区安全与健康辅助色。
- 信息层级明确：标题、辅助信息、状态和操作按钮使用稳定的字号与间距体系。
- 移动端优先：核心按钮高度不低于 88rpx，重要操作靠近拇指可触达区域。
- 状态可识别：审核中、未通过、已验证、未读通知均有独立色彩和文案。
- 深色模式同步：所有主页面使用全局设计变量适配深色模式。

## 已优化页面

1. 手机号登录：强化品牌氛围、隐私说明、授权状态和主操作按钮。
2. 首页信息流：新增内容型 Hero、推荐/关注分段控件、空状态和失败状态。
3. 动态卡片：优化作者信息、图片布局、话题标签、审核状态和互动区。
4. 发布页：按“正文—关联—图片”拆分任务，并增加底部固定发布栏。
5. 话题页：增加主题引导和层次更清晰的话题卡片。
6. 个人中心：重新组织用户信息、数据统计、快捷入口、服务菜单和个人动态。

## 全局规范

- 页面背景：`#f5f4f1`
- 主品牌色：`#ff6f4d`
- 辅助绿色：`#4f9f78`
- 卡片圆角：`30rpx`
- 主要按钮高度：`88rpx`
- 内容卡片间距：`22rpx`


## 发布页操作区

发布按钮采用页面底部正常内容流布局，不使用固定悬浮层。用户完成正文、关联信息、图片和发布提示后，滚动到页面末尾执行发布，避免遮挡表单内容及底部导航。
