# 测试用例 v1.0.0

## A. 自动检查

```bash
npm install
npm run verify
```

预期：结构、JSON、页面资产、云函数语法、TypeScript、契约测试和 Mock 集成测试全部通过。

## B. Mock 核心闭环

- B01 首次启动：生成演示用户、宠物、动态、关系、通知和管理员入口。
- B02 创建宠物：列表、详情和刷新后数据一致。
- B03 发布动态：宠物、话题、文字和图片完整展示。
- B04 `[待审核]`：仅“我的”可见，公共首页不可见。
- B05 模拟违规词：客户端和仓库均拒绝。
- B06 评论回复：显示“回复某用户”，父评论信息正确。
- B07 通知：逐条已读和全部已读后未读数正确。

## C. CloudBase 会话与权限

- C01 首次云登录：users 新增 OpenID 记录，role 默认 user。
- C02 再次启动：复用账号并更新 lastLoginAt。
- C03 disabled/deleted：主要云函数拒绝。
- C04 muted：允许浏览和点赞，不允许发帖、评论。
- C05 伪造宠物归属：createPost 返回 Forbidden pet。
- C06 删除他人内容：返回 Forbidden。
- C07 普通用户调用 admin：返回 Admin permission required。

## D. 云存储

- D01 用户头像、D02 宠物头像、D03 动态 9 图：保存 cloud fileID。
- D04 上传前压缩：支持压缩的图片使用压缩结果。
- D05 中途失败：已上传的本批文件被尝试清理。

## E. 内容安全

- E01 安全文字动态：approved。
- E02 图片动态：初始 pending，仅作者可见。
- E03 全部 pass 回调：approved 并通知作者。
- E04 任意 risky 回调：rejected 并尝试删除违规文件。
- E05 未配置回调：持续 pending，可在人工审核队列处理。
- E06 违规评论/举报明细：云函数拒绝，不产生记录。
- E07 人工拒绝后收到 pass 回调：仍保持 rejected。

## F. 社交与治理

- F01 关注/取消关注：关系和计数正确。
- F02 拉黑：移除双向关注，隐藏对方动态和评论。
- F03 拉黑关系关注：操作被拒绝。
- F04 举报帖子、评论、用户：reports 新增 pending 记录。
- F05 重复未处理举报：返回已有记录，不重复创建。
- F06 举报自己：被拒绝。

## G. 管理后台

- G01 Dashboard：用户、宠物、动态、pending 动态和 pending 举报计数正确。
- G02 举报处理：resolved/dismissed，举报者收到系统通知，生成 audit_logs。
- G03 人工审核：内容状态和作者通知正确。
- G04 用户禁言：到期时间保存，期间不能发布或评论。
- G05 用户停用/恢复：登录和业务访问符合状态。
- G06 当前管理员自停用：被拒绝。

## H. 隐私与账号

- H01 注销确认：需要二次确认。
- H02 云端注销：内容软删除、关系清理、资料匿名化、openid 替换。
- H03 注销后原身份重新访问：不会恢复已注销账号。
- H04 Mock 清除数据：重新进入恢复演示初始状态。

## I. 体验与兼容

- I01 首页首屏显示骨架屏，错误时可重试。
- I02 深色模式下文字、卡片和输入框可读。
- I03 减少动画系统偏好下不播放不必要动画。
- I04 超长昵称、正文、评论和 9 图无明显布局破坏。
- I05 iOS、Android 各至少一台真机完成全流程。


## 手机号强制登录

| 编号 | 场景 | 预期 |
|---|---|---|
| AUTH-01 | 首次普通打开 | 进入手机号登录页，无法看到首页内容 |
| AUTH-02 | 扫描动态详情小程序码 | 先进入登录页，授权成功后回到原动态详情 |
| AUTH-03 | 未勾选隐私说明 | 授权按钮不可点击 |
| AUTH-04 | 拒绝手机号授权 | 停留登录页，业务接口不可使用 |
| AUTH-05 | 授权成功 | users 写入 phoneBound、masked、countryCode、verifiedAt、phoneKey |
| AUTH-06 | 再次进入 | 已绑定账号直接进入目标页，不重复授权 |
| AUTH-07 | 游客/Mock 模式 | 显示模拟授权按钮，不读取真实手机号 |
| AUTH-08 | 主体或 AppID 无权限 | 展示权限配置提示，不进入社区 |
| AUTH-09 | 动态 code 重复消费或过期 | 授权失败并提示重新点击 |
| AUTH-10 | 同一手机号绑定另一账号 | 服务端拒绝并提示已绑定 |
| AUTH-11 | 绕过页面直接调用业务云函数 | 返回 PHONE_AUTH_REQUIRED |
| AUTH-12 | 注销账号 | 手机号摘要、脱敏值及验证状态被清除 |
